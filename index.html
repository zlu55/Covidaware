1. Introduction

1.1 Purpose

Since the beginning of COVID-19 information and statistics has been scattered across the internet. Resulting in a need for a website that simply displays information about COVID-19 in a user friendly manner.

This document outlines the proposed COVIDaware website. This document will outline the goals of our project, as well as outline the requirements we are trying to fulfil. This document outlines all features of our project, plans for interfaces, and design. This document is intended to be helpful to both users and developers.

1.2 Scope

The software product will be a user friendly website called COVIDaware. COVIDaware will be an intuitive method for accessing all information regarding COVID-19, in a user-friendly format. This website will not just display statistics, but will also be complemented with corresponding visual aids. The objective of COVIDaware is to create an all inclusive tool to help users develop a deeper understanding of COVID-19 data.

1.3 Definitions, Acronyms, and Abbreviations

GUI - Graphical User Interface, which will be the interface the user interacts with.
API - Application Programming Interface
COVID-19 - CO refers to Corona, VI refers to virus, D refer to disease, and 19 refers to the year the virus was found(2019)
Coronaviruses - refers to a large group of viruses, but under the context of COVIDaware, will be used to refer to COVID-19
Heat Map - represents data using colour on a map.

2. Overall Description

2.1 Product Perspective

COVIDaware is based on data from publicly available databases within Canada. Major components include SQL to store COVID data on a server using python or javascript to manipulate the data in a visually pleasant format for users. In addition, HTML and CSS will be used to create the web application.

2.2 Product Functions
  • Provide up to date data on COVID cases in Canada
  • Display heat maps indicating areas with a large amount of COVID cases
  • Display graphs to show the amount of cases and vaccinations throughout Canada
  • Display statistics specific to each province
  • Display a list of frequently asked questions based on province
  • Users can input postal code and get COVID information related to that area

2.3 User Characteristics

Our audience is intended for everyone in Canada who is concerned about the hotspots, possible areas of infection or what to do if covid-like symptoms occur. We want to provide this audience with up-to-date information about covid-19.

2.4 Constraints

There is no cost constraint for COVIDaware as it uses free software to implement the Web App. The project does have development constraints of time and reliability. The time constraint is that the project must be finished by the end of the Semester, April 12th. The reliability constraint is that the CovidWare Website doesn't refresh every time data has been updated in the Canada Covid database.

2.5 Assumptions and Dependencies

Requires the user to to have an internet connection. COVIDaware assumes that ( https://health-infobase.canada.ca/covid-19/ ), will be accurate and up-to-date. It is also assumed that the user has a device that is web accessible and is compatible with Google Maps API.
